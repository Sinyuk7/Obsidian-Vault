# 电影感冷蓝的密度与调色控制

> [!summary]
> 电影感冷蓝不是“把画面推蓝”，而是让蓝色同时服从光线逻辑、明度结构、饱和度分布和输出映射。厚重来自被控制住的 luminance 与 chroma，柔和来自 shadows、mids、highlights 之间的 tonal travel，自然来自 neutrals、暖色锚点、材质反射和空间方向的保留。

最实用的底层模型是：**暖的直射光 + 冷的环境 fill + 有选择的中性控制**。蓝色如果有明确的来源，例如 open sky fill、window ambience、twilight、haze、wet surface 或 practical contrast，就会读成 atmosphere；如果只是平均覆盖暗部和高光，就会读成 filter。

## 为什么蓝色会显得厚重

### Density 不是单纯饱和度

“厚”的蓝色通常不是画面里最亮、最跳的蓝，而是**饱和度与明度共同形成的色彩重量**。如果只提高 cyan saturation，而不控制 luminance，蓝色会变得明亮、轻、塑料，并像贴在画面表面。

Blackmagic 对 ColorSlice 的说明很适合解释这个现象：density 通过调整饱和色的 luminance 来模拟 subtractive film behavior，避免高饱和颜色变得不自然地明亮。实践上，这意味着蓝青色要有重量，常常需要在增加 chroma 的同时略微降低对应色相的 luminance，或用 Hue vs Lum、ColorSlice Density、Film Look Creator Richness 一类工具把蓝色“压进”明暗结构里。

> [!tip]
> 判断蓝色是否有 density，不要只看它够不够蓝；看最蓝的区域是否也过亮、发奶、脱离材质。如果答案是肯定的，问题通常在 luminance，而不在 hue。

### Tonal travel 决定柔和感

好的冷蓝不会从黑到白保持同一种强度。它会随着亮度区域改变性格：

| Tonal zone | 更自然的冷蓝行为 | 常见失败 |
|---|---|---|
| 极暗部 | 可以轻微偏冷，但需要限制 chroma noise 和压缩色块 | 黑位被染成高饱和蓝黑，噪点和 banding 被放大 |
| 暗部 | 可以更 dense，承载夜色、阴影和空间深度 | 暗部被整体抬亮，蓝色变薄、发灰 |
| 中间调 | 承担主要 atmosphere 和环境色 | 中间调与高光同样蓝，空间缺少层次 |
| 高光 | 逐渐减饱和，回到中性、白场或光源颜色 | 高光继续保持强 blue/cyan，显得电子、硬、digital |
| Specular / clipping zone | 尽量软化 roll-off，避免硬断层 | 极亮蓝色出 gamut、clip、solarization 或 hue shift |

ARRI 对 HDR 的提醒和 Cambridge in Colour 对 Curves 的说明指向同一件事：真实感来自柔和进入高光，而不是硬 clipping。ACES 2 这类 rendering transform 语境中强调的 lightness、colorfulness、hue 分离和高光 chroma roll-off，也是在处理同一个问题：颜色不能和亮度一起无限上升。

### 蓝色应该是一个家族，而不是一个色号

自然冷色场景很少只有一种精准 teal。更成熟的冷蓝通常是一个宽而连续的 hue family：

| Blue family | 典型观感 | 更适合的语境 | 风险 |
|---|---|---|---|
| Steel blue | 克制、城市、观察性、纹理感强 | 灰、金属、石材、低饱和服装、冷白 daylight | 过灰会变得死板、缺少情绪焦点 |
| Cyan blue | 空气感、水感、LED ambience、twilight | 天光、玻璃、湿路面、现代室内 | green 成分过多时补暖容易发绿 |
| Navy / deep blue | 重、夜间、低调、接近 deep blacks | low-key night、阴影、深色织物 | 暗部饱和过高会暴露噪点和压缩 |
| Blue-violet | 遥远、孤立、心理化、诗性 | 太空、极寒、梦境、孤独情绪 | 推得太远容易变成特效感或肤色污染 |
| Blue-green | 潮湿、荧光、病态或人工光 | 工业灯、低质 LED、地下空间 | 容易与黄绿色污染混在一起 |

这些不是科学定律，而是从业者的速记。真正重要的是：不同 tonal zone、材质和光源不应该全部被压成同一个 cyan target。

## 自然冷蓝来自光线、环境与感知

冷色调最容易自然化的方式，是在拍摄或设计阶段就建立可信的 light logic。蓝色阴影往往来自 open sky fill、atmospheric scatter、window light、twilight ambience 和环境反射；暖色 practicals、直射太阳、sodium lamp 或肤色则提供尺度、焦点和情绪对比。

| 现场因素 | 需要保留的差异 | 调色含义 |
|---|---|---|
| Sky-facing shadows | 朝向天空的暗面更容易偏 blue/cyan | 不要把所有阴影统一成同一色相；优先让可解释的 plane 变冷 |
| Bounce-filled shadows | 侧向墙面、地面或衣物会把自身颜色反射进阴影 | wall-facing、ground-facing、skin-facing 的暗部可以不同 |
| Warm practicals / sun | 暖色锚点解释冷色环境，并提供观看焦点 | 与其全局加暖，不如保留少量真正有用的 warm accent |
| Neutrals | 脏白、灰、黑、低饱和材质提供 reference system | 不要让每个 value region 都承担强 chroma |
| Materials | 皮肤、湿混凝土、玻璃、金属、哑光墙面反射方式不同 | 同一种 cyan cast 在不同材质上不应表现完全一样 |

Kodak 对 color adaptation、simultaneous contrast 和材质反射的说明很关键：观众对颜色的感知取决于相邻颜色、面积、位置、纹理和反射率。中性色不是“没风格”，而是让蓝色不喊出来的稳定器。它们告诉观众什么是白、什么被照亮、什么是情绪强调。

## 为什么补暖之后会发绿

“冷暖轴”和“绿洋红轴”不是同一条轴。Temperature 主要在 blue-amber / cold-warm 之间移动白点；Tint 或 CC 则处理 green-magenta。ARRI 和 Adobe 都把它们视为独立方向。

如果原来的冷色实际是 **cyan，也就是 green + blue**，只用加黄、减蓝或提高 temperature 来补暖，会先抵消 blue，green 却留在画面里，于是得到黄绿、橄榄绿或病态肤色。

> [!warning] 不要把所有“偏冷”都当成 temperature problem
> 荧光灯、部分 LED、钠灯和混合光常有不连续光谱或 green/magenta cast。此时需要检查 tint、minus-green、局部分区平衡和具体光源区域，而不是继续把整体色温推暖。

更稳定的修正顺序是：

1. 先判断污染是全局的，还是只存在于某个光源、材质或亮度区间。
2. 检查 green-magenta 轴，必要时先减 green 或加 magenta。
3. 如果 cyan 含绿过多，先把 cyan 往真正的 blue 方向整理。
4. 再决定是否增加 amber/warmth，或只保留局部 warm accent。
5. 最后回看 neutrals、skin、white fabric 和 practical highlights 是否仍有可信参照。

## “先染色，再控色”的准确含义

“先染色，再控色”不是鼓励一开始粗暴染蓝。更准确的意思是：**先看到最终色彩倾向，再针对最终产生的颜色问题进行约束**。

| 层级 | 目标 | 典型操作 | 检查点 |
|---|---|---|---|
| 技术层 | 让素材进入可预测状态 | Color Management / CST、Input Transform、曝光、白平衡、Tint、镜头匹配 | Log、Camera Gamut、Timeline Space、Output Transform 是否匹配 |
| Look 层 | 建立宽而柔的冷色方向 | 决定 steel blue、cyan-blue、navy、blue-green 或 violet-blue；低强度加入环境冷色 | 是否保留 neutrals 和 warm anchor |
| 密度层 | 让蓝色有重量 | Hue vs Lum、ColorSlice Density、Film Look Creator Richness、Selective Color、channel curves | 蓝色是否变厚，而不是只变鲜 |
| 过渡层 | 控制 shadows/mids/highlights 的饱和度分布 | Lum vs Sat、Blend If、soft keys、highlight desaturation、chroma roll-off | 高光是否自然减饱和；极暗部是否噪点增多 |
| 保护层 | 恢复空间和对象逻辑 | masks、windows、qualifiers、skin/practical/white protection、gamut compression | 人物移动是否闪烁；边缘是否硬；输出空间是否安全 |

如果在染色前就对原始颜色做大量狭窄限制，后面的整体 look 还会重新移动 hue、luminance 和 saturation，前面的控制经常失效。更可靠的方法是先完成技术校正，再建立大方向，最后约束最终画面里实际出现的问题。

## 工具映射

### Resolve / ACES 语境

| 方法 | 最适合解决的问题 | 风险 |
|---|---|---|
| Color Management / CST | 让输入、工作空间和输出转换一致，避免 Log/Gamut 操作异常 | transform 不匹配会让所有饱和度、色轮和 hue 操作表现失真 |
| Hue vs Hue | 整理 cyan、green、blue 的相互关系 | 推得太窄会造成色相断层 |
| Hue vs Sat | 削减过强的 cyan、blue、green 或 LED 色 | 过度削弱会让画面失去环境色 |
| Hue vs Lum / Density | 给蓝青色增加重量，避免薄亮 cyan | 压暗过多会让材质变脏或失去层次 |
| Lum vs Sat | 限制极暗部和高光的饱和度 | 曲线太硬会产生不自然分界 |
| Gamut Compression | 把极端颜色平滑压回可用范围，而非直接 clipping | 不能替代前面的 hue/lum/sat 设计 |
| Qualifiers / Windows | 保护肤色、灯光、天空、墙面、阴影 plane | 窄 key 或硬 mask 会产生闪烁、边缘和廉价合成感 |

### Photoshop 语境

| Photoshop 方法 | 最适合解决的问题 | 常见风险 |
|---|---|---|
| Curves on RGB | 重建 luminance structure 和整体 contrast，让蓝色有重量 | 重度曲线可能无意扭曲 hue/saturation 或让高光 clip |
| Curves on individual channels | 按 tonal zone 修正 casts，例如冷 shadows、中性 mids、干净 highlights | 过度使用会造成 crossovers、color breaks 或污染 neutrals |
| Selective Color | 在 broad look 后清理 cyan/blue、neutrals、whites、blacks 等 color families | 推得太远会让颜色像印上去，变脆、变假 |
| Color Balance | 在 shadows、mids、highlights 中建立 broad cool-warm intent | 不能单独解决 spectral green contamination |
| Gradient Map | 设计 shadows、mids、highlights 的连续 tonal palette | 100% 强度常显得 synthetic，通常需要 opacity、masking 或 blend mode restraint |
| Blend If | 按 luminance 限制冷色影响，保护 highlights 或 specular | 不拆分 slider 会有硬边 |
| Lab mode | 分离 lightness 与 color axes，在改 density 时减少不必要 hue shift | 如果不看 neutrals 和 skin，很容易过度 |
| Masks / clipping masks | 局部化 look，保留 face、practical、sky、foreground materials 的差异 | 糟糕 mask 是让 grade 显得像合成图的最快方式之一 |

一个简洁的记法是：**Curves 构建 density，Selective Color 清理 color families，Color Balance 设定 broad mood，Gradient Map 设计 tonal palette，Blend If 按 luminance 限制影响范围，Lab 分离 lightness 与 color axes，masks 恢复 spatial logic。**

## 常见失败模式

- **把蓝色当作 overlay，而不是 lighting consequence：** 场景里找不到 sky fill、window ambience、twilight、wet surface 或 practical contrast 来解释蓝色。
- **追逐 saturation，而不是 density：** cyan 很亮、很鲜，但没有重量，像表面滤镜。
- **移除所有中性色和 hue hierarchy：** 黑、灰、高光、practicals、肤色全部漂向同一个 cyan family，眼睛失去 reference system。
- **在错误轴线上修正冷暖：** 源头是 green contamination，却只用 amber/warmth 处理，结果变黄绿。
- **强迫 shadows、mids、highlights 穿过同一种 hue：** 高光仍高饱和，暗部也高饱和，缺少 tonal travel。
- **使用过窄 qualifier 或硬分区：** 人物移动时出现闪烁、边缘和色彩断层。
- **忽略 gamut 与输出空间：** 极亮蓝色、LED 和强 cyan 容易 clipping、hue shift、magenta solarization 或压缩色块。
- **错误 Color Management：** Input Transform、Timeline Space、Output Transform 不匹配时，任何调色工具都会表现异常。

## Review checklist

在判定一个 cold-blue grade 完成前，可以按顺序问：

1. 这个蓝是什么蓝：cyan、steel blue、navy、blue-violet，还是 blue-green？
2. 它有 density 吗，还是只是更鲜艳、更亮？
3. 它出现在哪些亮度区域？是否与 shadows、mids、highlights 的功能一致？
4. 高光有没有逐渐减饱和并回到中性、白场或光源颜色？
5. 画面是否还保留足够 neutrals、肤色段落或 warm accent 作为参照？
6. 不同 shadow planes 是否仍然不同，而不是全部被压成一个 cyan target？
7. 补暖之后，green-magenta 平衡是否仍然正确？
8. 蓝色是否与空间、光线、材质和物体结构绑定，而不是浮在表面？

真正柔和的电影蓝色，通常不是“蓝得少”，而是**蓝色的 hue、luminance、saturation、tonal roll-off 和 spatial masking 被同时控制住了**。
