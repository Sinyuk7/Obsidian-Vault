# Midjourney 参数设置指南
Last_Updated: 2026-03-06

## Overview [Required]
- Purpose: 提供 Midjourney 图像参考类与画布控制类参数的准确说明与最佳实践
- Scope: 涵盖 `--oref`、`--sref`、`--cref`、`--iref` 图像参考参数及 `--ar` 画布控制参数 / 不包含提示词语法、风格控制、质量参数等其他内容

## Core Concepts [Required]
- **语义理解**: Midjourney 对参考图像的认知方式，识别图像本质概念而非单纯复制像素
- **原型依从性**: 模型倾向于将参考对象还原为最常见状态的特性
- **注意力预算**: 模型处理提示词信息的资源限制，过多竞争元素导致细节丢失
- **图像参考参数**: 通过 URL 引入外部图像影响生成结果的参数类别
- **宽高比**: 控制画布尺寸比例的参数，影响构图与主体呈现

## Consolidated Principles [Required]

### 图像参考类参数

#### P1 [Critical]: `--oref` (Omni-Reference) 全能参考
- **功能**: 通过语义理解迁移参考图像中的角色、动物、建筑或物体特征，保持跨任务视觉一致性
- **语法**: `--oref <URL>`
- **配套参数**: `--ow` (Omni-Weight) 控制语义迁移强度，范围 1-1000，默认 100
- **权重策略**:
  - `--ow 400-1000`: 高权重，强化原型依从性，适用于需高度还原角色面部或物体结构的场景
  - `--ow 100`: 默认值，平衡特征提取与模型创造力
  - `--ow 10-50`: 低权重，削弱原型束缚，允许创造性重塑（如将靴子重塑为帽子）
- **三种提示词驱动模式**:
  1. **锚定稳定法**: 文本详细重申参考图标志性特征，明确指定新姿态或环境
  2. **放手交托法**: 文本仅提供极简描述（如"某人在银行排队"），完全依赖 `--oref` 填充细节，需配合较高 `--ow`
  3. **纯场景法**: 文本仅描述背景环境，不提及主体，模型自动将参考主体融入
- **画幅匹配原则**:
  - 参考图中的主体尺寸直接影响生成结果，需较小主体时应在参考图中预留大量空白
  - 生成图像的 `--ar` 必须与参考主体的物理形态匹配，高个子用竖屏，宽物体用横屏
- **特征选择限制**: 擅长捕捉宏观标志性特征（如风衣、特定发色），易丢失微小复杂细节（如带八个金字塔形宝石的银吊坠）

#### P2 [High]: `--sref` (Style Reference) 风格参考
- **功能**: 捕获参考图像的视觉风格（色彩、媒介、纹理、光线）并应用于新创作，不复制对象或人物
- **语法选项**:
  - `--sref <URL>`: 使用图像 URL
  - `--sref <数字>`: 使用 0 到 4294967295 之间的整数代码
  - `--sref random`: 随机返回风格代码（仅 V7 支持）
- **配套参数**: `--sw` (Style Weight) 控制风格影响强度，范围 1-1000，默认 100
- **多参考混合**:
  - 多个 URL: `--sref <URL1> <URL2> <URL3>`
  - 加权混合: `--sref <URL1>::2 <URL2>::1` 或 `--sref 123::1 456::2`
  - 顺序无关
- **权重关系原则**: 权重为相对比例，1.0:0.5 与 100:50 效果相同，可使用整数或小数
- **SRef 版本** (`--sv`):
  - V7: 共 6 个版本，`--sv 6` 为默认，`--sv 4` 为旧版模型
  - `random` 和代码仅兼容 `--sv 4` 和 `--sv 6`
- **常见问题处理**:
  - 参考图引入不需要的语义细节: 降低 `--sw`、修改提示词替换细节、使用权重强调/弱化元素

#### P3 [High]: `--cref` (Character Reference) 角色参考
- **功能**: 保持角色外观一致性，专注于人物特征而非风格
- **语法**: `--cref <URL>`
- **配套参数**: `--cw` (Character Weight) 控制角色权重
- **适用场景**: 跨提示词保持同一角色的面部、服装、姿态特征

#### P4 [Normal]: `--iref` (Image Reference) 图像提示
- **功能**: 通过图像 URL 前置影响构图和主体，作为提示词开头
- **语法**: `<URL> <文本提示词>`
- **配套参数**: `--iw` (Image Weight) 控制图像影响强度，范围 0-2，默认 1
- **与 `--oref` 区别**: `--iref` 更侧重构图迁移，`--oref` 侧重语义特征迁移

### 画布控制类参数

#### P5 [Critical]: `--ar` (Aspect Ratio) 宽高比
- **功能**: 控制画布尺寸比例，替代自然语言尺寸描述
- **语法**: `--ar <宽>:<高>`
- **常用比例**:
  - `--ar 1:1`: 方形，适用于头像、标志
  - `--ar 16:9`: 宽屏，适用于风景、电影感场景
  - `--ar 9:16`: 竖屏，适用于全身肖像、手机壁纸
  - `--ar 2:3`: 垂直比例，适用于全身人物
  - `--ar 3:2`: 横向比例，适用于标准照片
  - `--ar 3:4`: 垂直比例，适用于全身肖像
- **全身肖像强制要求**: 必须使用 `--ar 2:3`、`--ar 3:4` 或 `--ar 9:16` 等垂直比例
- **与 `--oref` 配合**: 宽高比必须与参考主体的物理形态匹配，否则导致比例失调或黑边

## Mechanisms & Logic [Conditional]

### 图像参考参数的工作机制
- **语义理解优先于像素复制**: `--oref` 识别图像本质概念，倾向于还原为最常见原型状态
- **原型陷阱**: 模型被参考物的语义原型困住时，需大幅降低 `--ow` 并强化文本锚定
- **注意力竞争**: 多个参考参数同时使用时，需调整各自权重避免某一参数过度主导

### 参数组合策略
- **保留 `--oref` 特征**: 降低 `--sw`、增加 `--ow`、降低 `--s`
- **保留 `--sref` 风格**: 增加 `--sw`、降低 `--ow`、降低 `--iw`
- **高依从性输出**: 降低 `--s` 至 50-75，添加 `--no <无意义词汇>` 提升正向提示词权重

## Constraints [Conditional]

### 硬性限制
- `--oref` 每次只能使用一张参考图
- `--ow` 取值范围 1-1000
- `--sw` 取值范围 1-1000
- `--ar` 比例需符合 Midjourney 支持列表

### 常见错误
- 使用 `--oref` 时宽高比与参考主体形态不匹配，导致黑边或比例失调
- 低 `--ow` 值仍无法打破原型束缚，需配合强烈文本锚定
- 复杂细节仅靠 `--oref` 无法保留，必须通过文本二次描述
