# AI 视频提示词构造规范与实战技巧

> [!info] Related Notes
> See [[02_Knowledge/AIGC/Z-Image/Z-Image Prompt Engineering 使用指南|Z-Image Prompt Engineering 使用指南]] for Z-Image constraints and [[02_Knowledge/AIGC/Governance/敏感词规避与提示词合规治理|敏感词规避与提示词合规治理]] for safety and governance framing.

## Summary

- 提供 AI 视频生成提示词的构造规范、优化方法与实战技巧。
- 覆盖从原始输入（文本意图 / 参考图片）到结构化提示词的完整转化流程。
- 包含七层通用架构标准、常见错误修正、跨模型迭代策略，以及 Seedance 和可灵等平台的特有操控技巧。
- 适用于所有基于扩散模型架构的主流 AI 视频生成工具。

## Applicability

- **适用于**：使用 AI 视频生成工具（文生视频 / 图生视频 / 多模态参考生成）进行内容创作的所有场景。
- **不适用于**：纯文本对话式 AI 交互；静态图像生成；与视频提示词无关的模型参数调优。

## Definitions

- **T2V (Text-to-Video)**：纯文本输入生成视频。
- **I2V (Image-to-Video)**：以静态图片为基础，结合文本提示词生成动态视频。
- **色彩锚点 (Palette Anchors)**：在提示词中显式声明 3–5 个具体色彩名称，防止多帧渲染中发生色彩漂移。
- **风格脊椎 (Style Spine)**：在多镜头系列生成中，通过逐字复用主体外貌、环境、灯光和色彩描述文本来维持视觉一致性的方法。
- **视觉锚点 (Visual Anchors)**：用具体可见的物理现象（如"暖色荧光灯"、"黄绿胶片感"）替代抽象主观形容词的策略。
- **动作原子化**：将复杂动作拆分为单一、不可再分的动作单元，每个提示词仅包含一个原子动作。
- **微提示词 (Micro-Prompts)**：将长叙事拆分为多个独立的 3–5 秒短片段提示词，分别生成后用剪辑软件拼接。
- **@ 绑定语法**：Seedance 平台中通过 `@素材名` 显式指定每个多模态输入的功能角色的语法机制。
- **运动笔刷 (Motion Brush)**：可灵平台中通过涂抹选区和方向箭头控制画面局部运动轨迹的工具。

## Requirements

> [!tip] R1: 七层架构优先级递减排列
> 提示词内容按以下七层顺序组织，模型注意力从上到下递减。不得颠倒层级顺序。

### Layer 1 — 主体定义与材质物理

- 禁止使用"一个人"、"一辆车"等宽泛词汇。
- 必须界定：外貌特征、年龄、体型、服饰的具体材质（尼龙、粗花呢、液态金属、玻璃等）。
- 材质描述决定后续光线反射率、布料重力下垂感和运动中的折痕变化。未定义材质的主体会呈现塑料质感。
- **示例**："一个留着胡须的 40 岁白人男性，穿着反光银色尼龙夹克"。

### Layer 2 — 运动学与时态原子化

- 使用单一的现在时态动词描述动作（"正在行走"、"转身"、"伸手"）。现在时态向模型传达持续状态，使其逐帧推演运动向量。
- 每个提示词仅包含一个原子动作。禁止使用连词（和、然后、接着）堆砌动作。
- 附加动作能量学副词控制节奏和力度："沉重地踱步"、"轻盈地跳跃"、"猛烈地摇晃"。副词影响动态模糊和重力表现。

### Layer 3 — 空间几何与环境交互

- 设定明确的时代和地点："1980 年代的东京霓虹街道"、"古老的石制祭坛"。
- 声明天气与大气介质（浓雾、大雨、漫天飞雪、灰尘微粒），为画面提供景深层次。
- 描述环境表面的物理状态："潮湿反光的柏油路面"、"粗糙的砂岩"、"布满划痕的金属桌"。表面属性决定环境光如何反射到主体上。

### Layer 4 — 摄影机语法与构图取景

- **景别控制**：
  - **大远景 (Extreme Wide Shot)**：强调广阔环境，主体极小。用于交代地理位置和气候。
  - **全景 (Wide Shot)**：主体全身入画，平衡环境与主体。适合宏大动作轨迹。
  - **中景 (Medium Shot)**：腰部或胸部以上。捕捉手部动作和面部情绪，保留适度背景。生成稳定性最高。
  - **特写 (Close-Up)**：专注细节和微表情。减少复杂背景的计算负担，提升细节保真度。
- **机位角度**：
  - **平视 (Eye-level)**：中性视角，适合写实纪录片风格。
  - **低角度 (Low Angle)**：增加主体压迫感和威严感。
  - **高角度 (High Angle)**：削弱主体力量，展示地面几何阵列。
  - **过肩镜头 (Over-the-shoulder)**：构建空间方位感和人物对话关系。

### Layer 5 — 摄像机运动学

- 所有摄像机运动保持缓慢和克制。剧烈摇摄 (Pan) 会撕裂边缘画面。
- **可用运动类型**：
  - **固定镜头 (Static / Locked-off)**：摄像机不动。主体动作复杂时，固定镜头确保画面不崩塌。
  - **推拉 (Dolly)**：摄像机前后移动。增强空间立体感。
  - **跟踪 (Tracking)**：摄像机平行于主体移动。
  - **手持 (Handheld)**：引入轻微摇晃，增加纪录片 / Vlog 真实感。
  - **稳定器 (Steadicam / Gimbal)**：运动丝滑，呈现工业电影质感。
- 专业运镜术语可直接使用：希区柯克变焦、一镜到底、低视角环绕、急停右摇、移轴变焦。

### Layer 6 — 光度学与美学色彩锚点

- 声明光源性质和方向："温暖的逆光 (Warm backlight)"、"柔和的边缘轮廓光 (Soft rim light)"、"刺眼的顶部荧光灯 (Harsh overhead fluorescent)"、"蝴蝶光 (Butterfly lighting)"。
- 引入电影光影流派：黄金时刻 (Golden Hour)、蓝调时刻 (Blue Hour)、明暗对比法 (Chiaroscuro)。
- 显式声明 3–5 个色彩锚点，防止色彩漂移。格式：`"Palette anchors: 深青色, 琥珀色, 胡桃木棕, 奶油白"`。

### Layer 7 — 技术规范、介质与独立音频

- **镜头焦距与光学特征**：
  - **35mm**：近似人眼纪实感。
  - **85mm 远摄**：压缩背景，适合人像特写。
  - **变形宽银幕镜头 (Anamorphic lens)**：引入横向镜头炫光和椭圆形光斑 (Bokeh)。
- **胶片与介质质感**：
  - 现代商业：`"8K, 极度清晰, 高动态范围"`。
  - 复古复刻：`"16mm 黑白胶片, Film grain, Chroma noise, Halation"`。
- **音频与对白分离封装**：音频指令不得混入画面描述。使用独立标签，放置在提示词最底部。对白极其简短，使用说话者标签（如"侦探："）。同时声明环境底噪（如"低沉的排气扇轰鸣声"）。

### R2: 提示词标准模板格式

```
[主体与材质]: [具体外貌特征, 年龄, 人种, 服饰的具体材质]
[动作与时态]: [单一现在时动词] + [动作幅度/节奏副词]
[环境与交互]: [地理位置与时代], [天气与大气介质], [特定表面的物理交互状态]
[机位与构图]: [明确的景别], [机位角度], [单一的运镜方向及设备质感]
[光度与色彩]: [光源方向与类型], [布光风格]。Palette anchors: [3-5 个具体色彩值]
[介质与光学]: [镜头焦距], [景深控制], [胶片质感与光学瑕疵]

Audio: [具体的环境底噪描述]
Dialogue:
[角色标签]: "[简短一句话对白]"
```

> [!caution] R3: 禁止的提示词写法（Anti-Patterns）
> - 禁止使用无法映射到像素的主观情感词（"悲伤的"、"史诗般的"、"感人的"、"电影感"）。替代方案：转化为具体的光学参数和构图指标。
> - 禁止在单个提示词中堆砌多个连续动作（"跑进房间 → 跳上沙发 → 开始叫"）。替代方案：仅描述最终状态或最关键的动作切面。
> - 禁止混淆文本问答与视频生成（"列出策略并生成视频"）。视频模型仅接收视觉描述指令。
> - 禁止在提示词中包含敏感数据（客户名称、API 密钥、内部文档）。使用通用占位符替代。
> - 禁止使用复杂从句和嵌套语法。使用极简短句和名词组合。
> - 禁止要求在 5–10 秒内完成跨越大时间尺度的变化（如"幼年成长为老年"），会导致画面崩塌。

### R4: 抽象概念的可视化转译规则

将所有主观形容词替换为具体的电影工业技术参数：

- `"悲伤的女人在雨中，电影感"` → `"中景特写 (Medium Close-up)，一个身穿深色风衣的女性在雨中缓慢行走，肩膀微垂。变形宽银幕镜头 (Anamorphic lens)，浅景深 (Shallow DOF)。色彩锚点：冷蓝色, 青灰色。高对比度明暗法 (Chiaroscuro)。"`
- `"一只狗跑进房间，跳上沙发，然后开始叫"` → `"定角镜头 (Locked-off shot)，一只金毛犬跳上棕色真皮沙发，动作迅速。室内布光。"`
- `"生成科技感的视频"` → `"广角镜头，一个现代科技感的数据中心，服务器指示灯闪烁，缓慢的轨道摄影 (Dolly track)。"`

### R5: 专业词汇替代日常用语

使用精确的摄影学词汇代替日常表达，更有效激发模型的高质量输出：

- 使用 `"Bokeh"` 而非 `"背景模糊好看"`。
- 使用 `"Tracking shot"` 而非 `"镜头跟着人走"`。
- 使用 `"Golden hour"` 而非 `"好看的夕阳光"`。
- 使用 `"Chiaroscuro"` 而非 `"明暗对比强烈"`。
- 使用 `"Dolly in"` 而非 `"镜头慢慢推近"`。

### R6: 数量描述使用泛化词汇

模型对确切数字缺乏精确计数能力。"海滩上有 10 只小狗" → "海滩上有一群小狗"。使用"一群、多只、几个"等泛化词汇。

## Procedure

### Step 1: 分析原始输入并确定生成模式

1. 判断输入类型：纯文本意图 → T2V 模式；已有参考图片 → I2V 模式；多素材组合 → 多模态参考模式。
2. **T2V 模式**：按七层架构完整填写所有层级。
3. **I2V 模式**：参考图片已锁定主体外观和环境基调，提示词重心转移至 Layer 2（动作）和 Layer 5（摄像机运动）。削减 Layer 1 和 Layer 3 中与图片重复的静态描述，防止文本与视觉编码器的信号冲突。
4. **多模态参考模式**：为每个素材显式分配功能角色（首帧 / 主体参考 / 动作参考 / 背景参考 / 音频参考）。

### Step 2: 拆解意图为原子化镜头单元

1. 将完整的叙事意图拆分为多个 3–5 秒的独立镜头单元（微提示词策略）。
2. 每个镜头单元遵守"单一镜头 + 单一动作"铁律。
3. 需要长序列时，分别生成多个高质量短片段，使用剪辑软件拼接。

### Step 3: 按七层架构组装提示词

1. 从 Layer 1 到 Layer 7 逐层填写内容。
2. 每一层使用具体可量化的技术参数，禁止使用主观形容词。
3. 将所有抽象概念转译为视觉锚点和光学指标。
4. 音频指令独立封装在提示词底部。

### Step 4: 跨模型迭代优化（5-10-1 法则）

1. **低成本验证**：在低参数 / 低分辨率模型上测试提示词的结构逻辑。重点观察主体动作、构图视角、色彩锚点是否生效。
2. **锁定变量迁移**：结构验证通过后，将完全相同的提示词文本逐字迁移至生产级高分辨率模型渲染。
3. **局部重混**：遇微小瑕疵时，禁止重写提示词（会重置随机种子生成完全不同的视频）。使用 Remix / Variation 功能，在锁定当前时空结构的前提下进行局部微调。

### Step 5: 构建风格脊椎维持多镜头一致性

1. 在同一系列的不同镜头中，Layer 1（主体）、Layer 3（环境）、Layer 6（光影色彩）、Layer 7（技术规范）的文本描述必须逐字保持一致。
2. 禁止随意更改光照或环境参数（如将"窗户透入的柔光与温暖的台灯辅光"改为"室内黄光"），否则模型会重新计算全局光照映射，导致前后镜头物理断裂。
3. 通过僵化复制粘贴光度学与色彩参数，锁定潜空间中相同的美学向量。

### Step 6: 质量评估

对每个生成的视频片段按以下四个维度验证：

1. **提示词依从性 (Prompt Adherence)**：焦距、色彩锚点、材质等硬性技术指标是否被精确执行。
2. **运动学质量 (Motion Quality)**：摄像机轨道是否平滑；人物骨骼动态是否自然；是否存在瞬移、肢体闪烁或鬼影。
3. **物理真实度 (Physical Realism)**：流体重力、阴影投射、材质反光是否符合物理法则。
4. **视听同步 (Audio Sync)**：唇语闭合是否与对白对齐；环境底噪是否与空间容积匹配。

## Exceptions

> [!warning] E1: I2V 模式下图片包含实体相框
> - **条件**：上传的参考图片带有明显的实体相框边框。
> - **处理**：裁剪掉相框再上传，否则模型会将其识别为"挂在墙上的照片"，生成静态照片墙的推拉效果而非主体动态化。

> [!warning] E2: 参考图片包含写实真人面部（Seedance 平台）
> - **条件**：上传包含写实真人脸部的素材。
> - **处理**：Seedance 系统自动拦截此类素材。替换为二次元、3D 模型、动物形象或使用面部遮挡物。

> [!warning] E3: 首尾帧差异过大
> - **条件**：首帧与尾帧的场景布局或色彩逻辑差异过大。
> - **处理**：模型放弃演算平滑补帧，生成硬切镜头效果。确保首尾帧在构图和色调上保持高度相似性。

> [!info] E4: 高复杂度物理碰撞场景
> - **条件**：需要渲染高频、高复杂的物理碰撞（高空抛物碎裂、连续球类弹跳等）。
> - **处理**：模型容易产生物理幻觉。转换为平滑的运动描述，降低物理交互的复杂度。

> [!info] E5: 纯文本无法精确控制局部运动轨迹
> - **条件**：文字描述无法让模型理解精确的空间运动路径（如"猫跳过盘子"被渲染为猫原地走动穿模盘子）。
> - **处理**：使用可灵平台的运动笔刷，在画面上涂抹选区并画出轨迹箭头；配合静态笔刷锁定不应移动的背景区域。

## Examples

### Example 1: 从模糊意图到结构化提示词（T2V）

- **Input（原始意图）**: "一个悲伤的女人在雨中行走，画面很有电影感"
- **Output（结构化提示词）**:
  ```
  [主体与材质]: 一个 30 岁亚洲女性，黑色长发湿漉漉地贴在脸颊，身穿深灰色羊毛大衣
  [动作与时态]: 缓慢地行走，肩膀微垂，目光低垂
  [环境与交互]: 夜间城市街道，大雨，潮湿反光的柏油路面，远处路灯光晕散射
  [机位与构图]: 中景特写 (Medium Close-up)，平视 (Eye-level)，缓慢跟踪 (Slow tracking)，Steadicam
  [光度与色彩]: 柔和的逆光 (Soft backlight) 来自远处路灯，蓝调时刻 (Blue Hour)。Palette anchors: 冷蓝色, 青灰色, 深炭色, 暖琥珀色
  [介质与光学]: 变形宽银幕镜头 (Anamorphic lens)，浅景深 (Shallow DOF)，轻微的镜头炫光 (Lens flare)

  Audio: 密集雨滴敲打路面的沙沙声，远处汽车驶过水坑的声响
  ```
- **Notes**: 原始意图中"悲伤"通过肩膀微垂、目光低垂等可视化身体语言表达；"电影感"通过变形宽银幕镜头、浅景深和明暗法等具体光学参数实现。

### Example 2: 图生视频提示词（I2V 降维重构）

- **Input**: 一张女性模特站在花田中的照片，需要生成动态视频
- **Output（结构化提示词）**:
  ```
  [动作与时态]: 轻盈地转身，裙摆随转动飘起
  [机位与构图]: 缓慢推进 (Slow dolly in)，从中景渐变至中近景
  [环境补充]: 微风吹动花朵轻微摇摆，花瓣零星飘落
  ```
- **Notes**: 主体外貌、服装材质、背景花田、基础光影已由参考图片锁定，提示词仅聚焦于动作和摄像机运动。禁止重复描述图片中已有的视觉信息。

### Example 3: Seedance 平台多素材融合（@ 绑定语法）

- **Input**: 需要将多张图片和一段参考视频融合生成一个视频
- **Output**:
  ```
  @图片1 作为首帧，角色形象参考 @图片2，打斗动作与运镜参考 @视频1，背景音乐使用 @音频1。
  参考 @图片2 的人物形象，完全复刻 @视频1 的所有运镜效果。镜头跟随主角在 @图片3 拐角奔跑，随后右摇 90 度拍摄 @图片4 的分叉路口。
  ```
