# Note Archetypes

Choose modules that answer the reader's actual question. Omit empty, redundant, or irrelevant modules. Do not copy a full template unless the user requests one.

| Archetype | Useful modules | Avoid |
| --- | --- | --- |
| Concept or evergreen | Definition, central claim, mechanism, boundaries, examples, related concepts | Project tasks or source logs without a reason |
| Research | Summary, question, findings, evidence, uncertainty, open questions | Unsupported finality |
| Source or literature | Source context, author claims, selected excerpts, interpretation, reusable ideas | Mixing source claims with personal conclusions without labels |
| How-to | Goal, prerequisites, steps, verification, failure modes | Generic background that hides the procedure |
| Project | Goal, scope, status, decisions, risks, next actions | Invented milestones or status values |
| Meeting | Participants, agenda, decisions, action items, unresolved topics | Rewriting discussion into unverified conclusions |
| Comparison | Criteria, options, evidence, conclusion, trade-offs | Tables used for long prose |
| MOC or index | Purpose, grouped links, navigation path, short annotations | Deep exposition duplicated from linked notes |
| Daily or capture | Timestamp, quick observations, tasks, raw ideas | Premature taxonomy or artificial summaries |
| Visual reference | Reference, observations, composition/light/color/pose cues, reusable prompts | Decorative image dumps without analysis |

For a substantive note, begin from the central question or conclusion. For a template, include only modules that will be filled repeatedly and label placeholders clearly.
